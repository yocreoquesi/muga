MUGA: Browser Test Suite

Manual validation checklist · Open this page with the extension active in Chrome

⚙ Setup

Environment: Code runs in WSL Ubuntu · Chrome runs on Windows host

Step 1: Load the extension in Chrome (once)
Open chrome://extensions → Enable Developer mode → Load unpacked → navigate to \\wsl.localhost\Ubuntu\home\yocreoquesi\projects\muga\src and select it.

Step 2: Serve this test page from WSL
In the WSL terminal run: npm run test:serve
Then open in Chrome: http://localhost:5555

Step 3: Open DevTools → Console to catch any runtime errors while clicking.

Important: Each test link must be clicked. Do not paste URLs in the address bar. The content script only intercepts click events.

Popup UI UI

Open the extension popup by clicking the MUGA icon in the Chrome toolbar.
  • Popup opens without JS errors in DevTools Console
  • Stats section shows 3 counters (tracking removed, affiliates active, referrals detected)
  • Counter values are numbers (0 or higher, not NaN, undefined, or blank)
  • "Enable / disable MUGA" toggle changes state and persists after popup is closed and reopened
  • "Inject my affiliate when there is none" toggle persists after close/reopen
  • "Notify me if there is a foreign referral" toggle persists after close/reopen
  • "Advanced preferences →" link opens the options page in a new tab
  • After clicking a link with tracking params (e.g. A-01), reopen popup. Counter for "tracking removed" should increment by 1

Context menu UI

Right-click on any link on this page to verify the context menu entry appears.
  • Right-click on any link below → context menu shows "MUGA: Copy clean link"
  • Click "MUGA: Copy clean link" on A-01 link → paste in address bar → should be https://example.com/ (no UTM)
  • Click "MUGA: Copy clean link" on A-05 link → paste → should have q= and page= but no utm_source
  • Click "MUGA: Copy clean link" on A-07 link (no params) → paste → URL unchanged

Global enable / disable UI

  • Open popup → disable MUGA (toggle off) → click A-01 → should land on the URL WITH utm_source (not cleaned)
  • Re-enable MUGA → click A-01 again → should land on the URL WITHOUT utm_source

Options page UI

  • Opens in a full tab (not a popup)
  • All 3 toggles (inject, notify, replace) reflect saved state
  • Toggle states persist after page reload
  • Blacklist: add an entry (e.g. example.com) → entry appears in list
  • Blacklist: remove the entry → list updates
  • Whitelist: same add/remove flow works
